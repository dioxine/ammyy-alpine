# Служба Ammyy Relay & Router для Alpine Linux

Сменить язык: 🇬🇧 [English Version](README.md)

---

Легковесный слой совместимости glibc для запуска бинарных файлов Ammyy напрямую в Alpine Linux с использованием высокооптимизированного, готового к продакшену минимального Docker-контейнера.

> [!NOTE]
> Чистый размер контента: **8.86 МБ** | Занимаемое место на диске: **29.9 МБ**

## Сетевая архитектура и карта портов

Стек работает в **режиме хост-сети (`network_mode: host`)**. Это полностью убирает лишнюю прослойку виртуального NAT-транслятора Docker, предотвращает задержки маршрутизации и решает проблему автоматической блокировки пакетов (anti-spoofing drop) подсистемой `nftables` на чистых серверах Alpine Linux.

Бинарные файлы привязываются напрямую к сетевым интерфейсам вашей реальной машины, используя следующие альтернативные порты по умолчанию:

| Порт     | Протокол | Компонент      | Назначение                                                      |
|:---------|:---------|:---------------|:----------------------------------------------------------------|
| **8080** | TCP      | `ammyy_router` | Основной рабочий порт роутера для клиентских подключений        |
| **8443** | TCP      | `ammyy_router` | Альтернативный рабочий порт роутера (вместо стандартного `443`) |
| **85**   | TCP      | `ammyy_router` | Встроенная веб-консоль управления роутера                       |
| **3080** | TCP      | `ammyy_relay`  | Рабочий порт реле (вместо стандартного веб-порта `80`)          |

> [!IMPORTANT]
> **Настройка локального IP для реле:** Если на вашем сервере поднято несколько виртуальных сетей или кастомных VPN-туннелей (например, WireGuard/Amnezia), адрес `0.0.0.0` может не привязаться корректно в режиме хост-сети. Откройте файл `config/relay.cfg` и вручную замените `0.0.0.0` на реальный физический LAN IP-адрес вашего сервера (например, `192.168.1.100`).

## Быстрое развертывание через Docker Compose

> [!IMPORTANT]
> Для управления демонами Docker и каталогами требуются права суперпользователя. Убедитесь, что вы запущены под учетной записью `root` или используете `sudo`.

### 1. Клонирование репозитория
Склонируйте репозиторий напрямую в каталог `/opt`, чтобы сразу развернуть всю готовую структуру файлов:

```bash
git clone -b docker https://github.com/dioxine/ammyy-alpine.git /opt/ammyy
```

### 2. Запуск стека служб
Перейдите в созданную рабочую папку и поднимите микро-контейнер в фоновом режиме:

```bash
cd /opt/ammyy
docker compose up -d
```
*Docker самостоятельно развернет оптимизированный образ `gdioxine/ammyy-stack:latest` с Docker Hub, подключив его напрямую к физической сети хоста без создания виртуальных сетевых мостов.*

> [!NOTE]
> **Сноска для Alpine Linux:** Если при первом запуске внешние клиенты получают таймаут подключения, убедитесь, что остаточные правила файрвола сброшены до чистого состояния командами `nft flush ruleset` или `iptables -F FORWARD`.

## Структура рабочей директории

Приложение изолирует исполняемую среду внутри слепка контейнера, оставляя на хост-машине только текстовые конфигурации и логи:

```text
/opt/ammyy/
├── docker-compose.yml     # Файл оркестрации стека служб
├── config/                # Изменяемые файлы конфигураций
│   ├── relay.cfg          # Сетевые настройки и порты реле
│   └── router.cfg         # Активный конфигурационный файл роутера
└── logs/                  # Хранилище оперативных логов приложения
    ├── failed.log         # История неудавшихся подключений к роутеру
    ├── ld-linux.so.2.log  # Лог отладки 32-битного интерпретатора
    ├── logins.log         # Аудит успешных авторизаций клиентов
    ├── relay.log          # Главный системный журнал работы реле
    └── sessions.log       # Статистика активных сессий роутера
```

## Управление жизненным циклом контейнера

Контролируйте фоновые процессы с помощью стандартных команд Docker Compose внутри папки `/opt/ammyy`:

*   **Просмотр логов в реальном времени:** `docker compose logs -f ammyy-app`
*   **Применение измененных настроек:** `docker compose restart ammyy-app`
*   **Полная остановка и удаление ресурсов:** `docker compose down`
